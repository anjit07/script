# Development Rules

## Code Style
- use constructor injection
- use final where possible
- use lombok minimally
- keep methods focused
- max one responsibility per class
- prefer composition
- avoid deep nesting
- avoid duplicated code

## Spring
- service contains business logic
- controller handles request only
- repository handles persistence only
- use dto mapping
- validate inputs
- use transaction at service layer

## Output
- generate compile ready code
- keep explanation minimal
- update existing code first
